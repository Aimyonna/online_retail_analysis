# 跨境电商零售数据全链路挖掘与客户价值分析

## 项目背景
本项目基于真实的数十万条跨国在线零售订单流水数据。由于原始数据存在大量退货记录（负数）、缺失值及未分类商品，直接进行业务分析难度较大。
本项目旨在通过端到端的数据清洗、特征工程与机器学习算法，挖掘隐藏在庞大流水账背后的“高价值客户画像”与“商品捆绑销售黄金法则”，从而为业务端的**精准营销、客户召回及客单价提升**提供数据策略支撑。

## 技术栈与工具
* **语言与分析库:** Python 3.10+
* **机器学习与数据挖掘:** Scikit-Learn (K-Means, StandardScaler), MLxtend (Apriori)
* **核心业务模型:** RFM 客户价值模型, 购物篮关联分析

## 核心项目结构
```text
online_retail_analysis
 ┣ data                       # 数据目录
 ┃ ┗ clean_retail.csv         # 清洗后的业务特征数据
 ┃ ┗ archive                  # 原始数据

 ┣ rfm.py                     # RFM 特征提取

```

## 核心业务模块与商业洞察

### 模块一：RFM 商业模型特征提取 (rfm.py)
* **Action:** 导入洗净的电商订单数据，利用 Pandas 的 `groupby` 与 `agg` 组合拳，将庞大的底层流水账按 `CustomerID` 进行高维聚合。一键提炼出每位独立客户的 Recency（最近一次消费天数）、Frequency（不重复订单频次）和 Monetary（累计消费总金额）三大核心商业指标。
* **Result:** 成功将 4338 位独立客户的“底牌”和消费特征数字化，完成了从“交易数据”向“用户画像特征”的跨越，为后续的机器学习算法打下了最坚实的数据地基。

### 模块二：K-Means 机器学习客户智能聚类 (kmeans.py)
* **Action:** 调用 `rfm.py` 计算出的特征矩阵，使用 `StandardScaler` 实施统计学标准化，彻底消除天数、次数和金额之间的量纲差异；随后引入无监督学习 `K-Means` 算法（ `n_clusters=4`, `random_state=42`），让机器进行多维空间距离计算，实现自动化的客户价值分层。
* **商业洞察与精准运营策略:**
  * **核心金主:** 人数极少（仅 13 人），但人均消费断层式领先（十几万英镑），且近期非常活跃。**策略:** 1V1 专属客服伺候，配置大客户最高优先级权益，定期透传新品，严防流失。
  * **高价值忠诚客户:** 复购频次高、消费力强，是平台的中流砥柱。**策略:** 推出 VIP 会员阶梯积分体系，用“满减/满赠”等高客单价活动刺激其向超级金主转化。
  * **大众基石客户:** 人数最多（占比超 60%），购买习惯和活跃度普通。**策略:** 精准营销主战场，定向推送爆款引流品与限时折扣券，重点拉升其复购频率。
  * **高危流失客户:** 平均已大半年没来消费，客单价极低。**策略:** 触发自动化流失召回机制（如发送专属大额红包），若召回 ROI 持续为负，则战略性放弃。

### 模块三：Apriori 购物篮关联规则挖掘 (apriori_analysis.py)
* **Action:** 抽取特定区域（法国区）订单构建 One-hot 购物篮二值化矩阵；应用 `mlxtend` 库的 `Apriori` 算法，严格设定最小支持度（`min_support=0.05`）及提升度（`min_threshold=1`）阈值，高效过滤随机噪音，挖掘出高置信度的商品强关联规则。
* **商业洞察与流量变现策略:**
  * 成功提取出协同效应最强的 Top 10 商品捆绑销售黄金规则（发现买 A 极大促进了买 B，Lift 提升度远大于 1）。
  * **策略:** 赋能电商运营部门，在商品详情页或结算页一键上线“猜你喜欢”交叉推荐算法；或直接将强关联的“高毛利+低毛利”商品打包成“组合优惠包”进行售卖，以最低的获客成本直接拉升客单价。

## 如何运行本项目
克隆本项目到本地机器。

在终端运行 pip install -r requirements.txt 安装所需依赖库。

确保清洗后的数据文件就位，按顺序执行 rfm.py -> kmeans.py -> apriori_analysis.py 即可复现全部业务分析结果。
